My Project

This site has been developed using Python and the Django framework. This choice was influenced by my experience at Capula where python is the primary language used by developers.

The reason I chose to develop a website is because I have previously completed a JavaScript website coding project with Multiverse. Therefore, I could focus more on Python as I already have experience in HTML and CSS, as well as a basic understanding of SQL.

What is the purpose of this project?

This project is designed to assist our infrastructure team in managing requests. Since these requests require manager approval, the website provides an efficient way for the manager and the rest of the team to review and respond to them.

How to use my site

My site can be used by creating an account, which will then be stored in the database. Once logged in, users will be able to view and create tickets. A GET request from the database will then display the username on the ticket.

In the 'Tickets' page you can get an overview of all tickets. The idea is to create an easy to navigate platform so that tickets can be quickly filtered through.

The 'Create Ticket' form is user friendly, so it can be used across different projects and scenarios. It allows users to input key details as well as an image.

Finally, the 'Admin' page allows users who have certain permissions to view and create tickets, but, also update or delete them. They also have access to view all accounts which is very useful for our infrastructure team.

Where could I take this in the future?

In the near future I hope to add some more quality of life features such as a search bar, a way to filter through tickets and update/delete buttons on the ticket view page allowing users to edit tickets they have raised.

If I was to continue to work on this in the future, I would like to add a way so logged in users can only see their own tickets, unless they have admin permissions which would allow them to see the complete overview.

There are a few more quality of life features that could improve the application, these could be a history overview of all tickets and a way to generate a report on how many tickets have been created, denied and accepted over a period of time and what they contained. However, I aim to move on and develop better and more complex appplications or other programs to expand my portfolio.
